PROJECT PIPELINE
================

GOAL:
Predict future body-relative attacker motion / pose / ball-relative movement
from past broadcast football footage.


--------------------------------------------------
1. DATA COLLECTION
--------------------------------------------------

- Collect football dribbling / carry / 1v1 clips
- Prefer:
    - real-speed footage
    - similar camera style
    - attacker-centered clips
- Avoid:
    - heavy slow motion
    - extreme camera cuts
    - occluded players

Suggested amount:
    150–500 clips initially


--------------------------------------------------
2. CLIP LABELING / METADATA
--------------------------------------------------

For each clip:
    - attacker identity
    - clip start/end
    - optional action label:
        dribble / carry / cut / pass / shot

Optional:
    dominant foot
    player role
    left/right attack direction


--------------------------------------------------
3. POSE EXTRACTION
--------------------------------------------------

Use:
    - YOLO Pose
    - MediaPipe Pose
    - OpenPose
    - ViTPose

Extract keypoints per frame:
    shoulders
    hips
    knees
    ankles
    elbows
    feet
    etc.

Frame rate:
    ~15 fps is reasonable


--------------------------------------------------
4. BALL EXTRACTION
--------------------------------------------------

Detect or label soccer ball position:
    (x_ball, y_ball)

Methods:
    - manual labeling
    - YOLO ball detector
    - semi-manual correction


--------------------------------------------------
5. NORMALIZATION
--------------------------------------------------

Goal:
remove camera movement / zoom effects

Use:
    hip center as origin

Example:
    p'_i = p_i - p_hipcenter

Scale by:
    body height
    shoulder width
    torso length

Example:
    p''_i = (p_i - p_hipcenter) / body_scale

Optional:
    rotate coordinates into body frame


--------------------------------------------------
6. FEATURE ENGINEERING
--------------------------------------------------

Compute:
    - shoulder orientation
    - hip orientation
    - torso angle
    - angular velocity (omega)
    - pose change between frames
    - ball-relative position
    - ball-foot distance
    - body-motion alignment
    - limb angle changes

Example:
    omega = delta(theta)/delta(t)

Possible representations:
    pose graph
    normalized keypoints
    temporal pose sequence


--------------------------------------------------
7. DATASET CONSTRUCTION
--------------------------------------------------

Example:
    past 30 frames -> next 5 or 15 frames

Input:
    pose/features from frames t-29 ... t

Output:
    future pose/features from frames t+1 ... t+k

Representation:
    vectors / tensors


--------------------------------------------------
8. MODEL TRAINING
--------------------------------------------------

Start simple:
    - linear baseline
    - constant-angle baseline
    - MLP
    - LSTM
    - GRU

Later:
    - Temporal CNN
    - Transformer
    - ST-GCN
    - Graph Neural Networks


--------------------------------------------------
9. EVALUATION
--------------------------------------------------

Do NOT use only classification accuracy.

Use:
    - MSE
    - MAE
    - angle prediction error
    - keypoint displacement error
    - trajectory deviation
    - future pose similarity

Compare against:
    - constant motion baseline
    - no-motion baseline


--------------------------------------------------
10. EXTENSIONS
--------------------------------------------------

Later additions:
    - nearest defender
    - pressure
    - teammate spacing
    - multi-player interaction
    - tactical context
    - full trajectory prediction
